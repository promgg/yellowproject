# MJ-Admin

แผงควบคุมแอดมินของเซิร์ฟเวอร์ เปิดด้วยปุ่ม **PgUp**

---

## เวลา & สภาพอากาศ

อยู่ในแผงหลัก (ไม่ต้องเลือกผู้เล่น) — หมวด **"เวลา & สภาพอากาศ"**

| ปุ่ม | ทำอะไร |
|---|---|
| ตั้งเวลา | ใส่เวลาแบบ `HH:MM` (24 ชม.) |
| หยุด/เดินเวลา | สลับหยุดเวลาไว้กับปล่อยให้เดินต่อ |
| เปลี่ยนอากาศ | เลือกจากดรอปดาวน์ 21 แบบแล้วกด |
| ล็อก/ปลดล็อกอากาศ | ล็อก = ค้างอากาศปัจจุบัน / ปลด = คืนให้เปลี่ยนเองตามพยากรณ์ |
| คืนค่าอัตโนมัติ | คืนทั้งเวลาและอากาศเป็นอัตโนมัติพร้อมกัน |

### ข้อควรรู้

**สั่งผ่าน `weathersync` เท่านั้น ห้ามใช้ native** — `weathersync` มีลูป broadcast เวลา/อากาศให้ทุก client **ทุก 5 วินาที** ถ้าไปตั้งเองด้วย `SetWeatherType`/`NetworkOverrideClockTime` จะโดนเขียนทับหายภายใน 5 วิ ดูเหมือนปุ่มเสีย

**เลือกอากาศแล้วระบบล็อกให้อัตโนมัติ** — ถ้าไม่ล็อก ลูปพยากรณ์ของ `weathersync` จะหมุนอากาศเปลี่ยนเองภายใน 1 ชั่วโมงในเกม สั่งแล้วอยู่ไม่ทน กดปุ่ม "คืนค่าอัตโนมัติ" เมื่อต้องการปล่อยคืน

**ต้อง `ensure weathersync` ก่อน** ไม่งั้นปุ่มจะไม่ทำงานและขึ้น log เตือนในคอนโซล

## รีเซ็ตบัญชีผู้เล่น

> ⚠️ **ลบข้อมูลถาวร** — ทุกตัวละคร ม้า ของ เงิน ตู้เซฟ ทั้งบัญชี
> ผู้เล่นจะถูกเตะออก และเข้ามาใหม่เป็นผู้เล่นใหม่ที่ต้องสร้างตัวละครใหม่ทั้งหมด

**วิธีใช้:** เลือกผู้เล่นในรายชื่อ → เลื่อนลงล่างสุดถึงหมวด **"โซนอันตราย"** → กด **รีเซ็ตบัญชี** → **พิมพ์ชื่อตัวละครให้ตรงเป๊ะ** เพื่อยืนยัน

พิมพ์ชื่อไม่ตรง = ยกเลิกทันที ไม่มีอะไรถูกลบ (คำสั่งอื่นในเซิร์ฟใช้ "พิมพ์ 1" ซึ่งเบาเกินไปสำหรับงานที่กดพลาดครั้งเดียวข้อมูลหายทั้งบัญชี)

**สิทธิ์:** `Config["Perms"].admin.CanResetAccount` · รีเซ็ตตัวเองไม่ได้

### สำรองอัตโนมัติทุกครั้ง

ก่อนลบ ระบบจะอ่านข้อมูลของบัญชีนั้นทั้งหมดออกมาเขียนเป็นไฟล์:

```
MJ-Admin/backups/reset_steam_11000010d615891_20260719_213045.json
```

**ถ้าสำรองไม่สำเร็จ ระบบจะยกเลิกการลบทั้งหมด** ไม่ลบสักแถว

## 🔄 คืนของให้ผู้เล่น (กู้คืนจากไฟล์สำรอง)

### ขั้นตอน

**1. ให้ผู้เล่นออกจากเซิร์ฟก่อน** ⚠️ สำคัญที่สุด

ถ้าผู้เล่นยังออนไลน์ `vorp_core` จะถือตัวละครไว้ใน memory และเซฟทับตอนออก — ของที่เพิ่งกู้กลับมาจะหายอีกรอบ (คำสั่งเช็คให้แล้ว ถ้ายังออนไลน์จะไม่ยอมทำงาน)

**2. หาชื่อไฟล์สำรอง**

อยู่ใน `resources/[MJDEV]/MJ-Admin/backups/` ชื่อไฟล์มี identifier กับวันเวลาอยู่แล้ว หรือดูจาก log ตอนที่รีเซ็ตก็ได้

**3. สั่งจาก console ของเซิร์ฟเวอร์** (txAdmin live console ก็ได้ — **ไม่ใช่** F8 ในเกม)

```
mjadmin_restore reset_steam_11000010d615891_20260719_213045.json
```

**4. อ่านผลในคอนโซล**

```
[MJ-Admin/restore] กำลังกู้คืน steam:11000010d615891 (สำรองเมื่อ 2026-07-19 21:30:45)
  users                            กู้ 1/1 แถว
  characters                       กู้ 1/1 แถว
  character_inventories            กู้ 37/37 แถว
  items_crafted                    กู้ 37/37 แถว
  player_horses                    กู้ 3/3 แถว
  ...
[MJ-Admin/restore] เสร็จ — กู้สำเร็จ 99 แถว, ล้มเหลว 0 แถว
```

**5. ให้ผู้เล่นเข้าเซิร์ฟใหม่** ตัวละครกับของจะกลับมาครบ

### ข้อจำกัดที่ต้องรู้

**กู้ได้เฉพาะสิ่งที่อยู่ใน DB** — โค้ดรับรางวัลที่เคยกด (`MJ-CodeReward`) ถูกล้างตอนรีเซ็ตและ**ไม่ถูกกู้กลับ** ผู้เล่นจะกดโค้ดเดิมได้อีกครั้ง ถ้าไม่ต้องการต้องแก้ `MJ-CodeReward/core/used_codes.json` เอง

**ถ้าผู้เล่นเล่นต่อหลังรีเซ็ตแล้วค่อยกู้** จะมีตัวละครใหม่กับตัวละครเก่าอยู่พร้อมกัน และแถวที่ `id` ชนกันจะขึ้น "ล้มเหลว" (ใช้ `INSERT IGNORE` กันข้อมูลใหม่โดนทับ) — กู้เร็วที่สุดหลังรู้ตัวว่าลบผิดจะสะอาดที่สุด

**สั่งซ้ำได้ปลอดภัย** — `INSERT IGNORE` แถวที่กู้ไปแล้วจะถูกข้าม ไม่เกิดของซ้ำ

**ไฟล์สำรองไม่ถูกลบอัตโนมัติ** — สะสมไปเรื่อยๆ (ไฟล์ละไม่กี่ KB) ลบเองได้เมื่อแน่ใจแล้วว่าไม่ต้องใช้

## สิ่งที่ระบบรีเซ็ตลบ / ไม่ลบ

### ลบ — ผูกกับตัวละคร (ทุกตัวละครในบัญชี)

ของในกระเป๋า · ไอเทม crafted · อาวุธ · fastslot · ตั้งค่ากระเป๋า · ธนาคาร+ตู้เซฟ · ม้า (VORP + BCC) · เกวียน · รถไฟ · ชุด · บัตรประชาชน · เมือง+เชื้อสาย · ฟาร์มสัตว์ · ตลาดกลาง (ที่เป็นคนขาย) · อันดับทั้ง 6 ตาราง · คุก · บริการชุมชน · บ้าน/ห้อง/สมุนไพร/น้ำมัน

### ลบ — ผูกกับบัญชี (steam)

users · whitelist · battlepass (ทั้ง 2 ระบบ) · เช็คอินรายวัน · เควสรายวัน · โค้ดรับรางวัล · รูปในโทรศัพท์ · สถานะหิว/กระหาย · แอนิเมชันที่บันทึกไว้ · playtime · ban mic

### ไม่ลบ — ตัดลิงก์แทน

| ตาราง | เหตุผล |
|---|---|
| `lp_marketplace.buyer_id` | อยู่ในแถวขายของผู้เล่น**คนอื่น** ลบทิ้ง = ไปลบประวัติขายของเขา → `SET NULL` |
| `nx_graverobbery_graves` | เป็น world-state (คูลดาวน์หลุมศพ) ไม่ใช่ของผู้เล่น → `SET NULL` |

### ไม่แตะเลย — ยืนยันรูปแบบค่าไม่ได้

`mjdev_board` · `rf_topics` / `rf_replies` (เว็บบอร์ด) · `mjdev_telegrams` · `mjdev_saveplayertelegram`
ทั้งหมดว่างตอนที่ตรวจ DB จึงยืนยันไม่ได้ว่าคอลัมน์เก็บ identifier หรือเบอร์โทร — เดาผิดแล้วไปลบของคนอื่น ต้องลบเองถ้าจำเป็น

**`mjdev_backpack` ไม่มีคอลัมน์ผูกตัวละครเลย** (มีแค่ `id`/`backpackid`/`model`/`limit`) แถวเป้จึงค้างใน DB เสมอ — เป็นแบบนี้มาก่อนมีฟีเจอร์นี้แล้ว

## หมายเหตุทางเทคนิค

**ทั้ง DB ไม่มี `FOREIGN KEY` / `ON DELETE CASCADE` เลยสักตัว** และ `DeleteCharacter` ของ `vorp_core` ลบเฉพาะแถวใน `characters` — ข้อมูลที่เหลือค้างเป็นขยะถาวร ระบบรีเซ็ตนี้จึงต้องไล่ลบเองทีละตาราง

ตอนตรวจ DB จริงพบ **ข้อมูลค้าง 117 แถวจากตัวละคร 9 ตัวที่ถูกลบไปนานแล้ว** ยืนยันว่าปัญหานี้เกิดขึ้นจริง

**ต้องเตะผู้เล่นออกเสมอหลังรีเซ็ต** ไม่งั้น `vorp_core` เซฟตัวละครเก่าจาก memory กลับลง DB ตอนออก = แถวที่เพิ่งลบโผล่กลับมา
