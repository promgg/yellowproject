# lp_planting

ระบบปลูกพืช — เขียนใหม่แทน `MJ-Planting`

## การเล่น

ใช้เมล็ดในเขตไร่ → ต้นกล้าลงตรงหน้าทันที → กดค้าง **E** ที่ต้นเพื่อ **ใส่ปุ๋ย** → **รดน้ำ** → รอ **20 นาที** → **เก็บเกี่ยว**

- ปลูกได้ **10 ต้นต่อไร่ต่อตัวละคร**
- ต้นอยู่ได้ **24 ชั่วโมงนับจากตอนปลูก** ไม่มาทำให้ครบจะถูกลบทิ้ง
- **เห็นเฉพาะต้นของตัวเอง** คนอื่นมองไม่เห็นและแตะไม่ได้
- ต้องมี **ปุ๋ย** (`compost`) และ **ถังน้ำ** (`tool_bucket`) — ถังเติมได้ที่จุดปั๊มน้ำในไร่ เติมครั้งละ 10 ครั้ง

## ต่างจาก MJ-Planting อย่างไร

| | MJ-Planting | lp_planting |
|---|---|---|
| เก็บข้อมูล | memory ล้วน | **DB** — รีสตาร์ทเซิร์ฟต้นยังอยู่ |
| ออกเกมแล้วกลับมา | **ต้นหาย เมล็ดเสียฟรี** | ต้นอยู่ครบ |
| นาฬิกา | `GetGameTimer()` รีเซ็ตทุกรีสตาร์ท | `os.time()` เวลาจริง โตต่อแม้เซิร์ฟปิด |
| โควตา 10 ต้น | มีตัวเลขใน config แต่**ไม่เคยเช็ค** | บังคับจริงฝั่ง server |
| อ้างพืชด้วย | เลขลำดับใน config | **ชื่อเมล็ด** |
| จุดเติมน้ำ | โซนละ 1 จุดตายตัว | ใส่กี่จุดก็ได้ใน config |

## ติดตั้ง

**1. รัน SQL** (จำเป็น ไม่งั้นปลูกไม่ได้)

```sql
source resources/[LP]/lp_planting/sql/lp_planting.sql
```

**2. ปิดตัวเก่า** ใน `MJDEV.cfg` — สองตัวลงทะเบียนเมล็ดชื่อเดียวกัน ถ้าเปิดพร้อมกันจะทับกัน

```
# ensure MJ-Planting
```

**3. เปิดตัวใหม่** ใน `MJDEV.cfg` หรือ cfg ที่เหมาะสม

```
ensure lp_planting
```

## จุดเติมน้ำ

**เมืองละ 4 จุด ครบทั้ง 3 เมืองแล้ว** พิกัดเดินเก็บจากในเกมจริง ทุกจุดอยู่ในรัศมีไร่

| เมือง | ไกลสุดจากกลางไร่ | ใกล้กันสุด |
|---|---|---|
| Valentine | 29 m | 6.6 m |
| Annesburg | 33 m | 19 m |
| Rhodes | 54 m | 27 m |

เพิ่ม/ย้ายได้ใน `config.lua` → `Config.Zones[*].waterPoints` โค้ดรองรับกี่จุดก็ได้

```lua
waterPoints = {
    { coords = vector3(x, y, z), heading = h },
},
```

**z ไม่ต้องเป๊ะ** — prop สแนปลงพื้นเองตอน spawn ขอแค่ x/y ไม่ไปชนหิน/กำแพง

**หมายเหตุ Rhodes:** ไร่นี้เป็นแถบยาวตามแกน y (~107 m) ไม่ใช่วงกลม จุดกลางเดิมอยู่ปลายแถบ
ทำให้จุดเติมน้ำหลุดนอกเขต จึงย้ายจุดกลางมากึ่งกลางแถบ — **blip กับเขตปลูกขยับไปทางเหนือ ~54 m จากของเดิม**

## ปรับค่า

ทุกอย่างอยู่ใน `config.lua`:

| ค่า | ตอนนี้ |
|---|---|
| `MaxPlantsPerZone` | 10 ต้น/คน/โซน |
| `PlantTimeoutHours` | 24 ชม. (0 = ไม่หมดอายุ) |
| `CropDefaults.growSeconds` | 1200 (20 นาที) |
| `CropDefaults.swapSeconds` | 600 (สลับเป็นโมเดลต้นโตตอนครึ่งทาง) |
| `WaterRefill.usesPerRefill` | 10 |
| `BanditChance` | 40% (0 = ปิด) |

เพิ่มพืชใหม่ = เพิ่ม entry ใน `crops` ของโซนนั้น ระบบลงทะเบียนไอเทมให้เอง

## หมายเหตุสำหรับคนแก้โค้ดต่อ

**server ตัดสินทุกอย่าง** — client ขอได้อย่างเดียว ทุก RPC ตรวจซ้ำหมด: เจ้าของ, ระยะ, ลำดับขั้น, มีของจริงไหม, โตครบเวลาจริงไหม, cooldown

**เวลาที่ส่งให้ client เป็น "ผ่านมากี่วินาทีแล้ว" ไม่ใช่ timestamp** — client ไม่มี `os.time()` ให้เทียบ มีแต่ `GetGameTimer()` ที่นับจากตอนเปิดเกม ถ้าส่ง timestamp ดิบไปให้ client ลบเอง จะได้เลขมั่ว (บั๊กที่ MJ-Planting เดิมเป็นอยู่)

**หักของแล้วต้องคืนถ้าบันทึกไม่สำเร็จ** — ตอนปลูก หักเมล็ดก่อน `INSERT` ถ้า insert พังจะคืนเมล็ดให้

**ต้นของคนที่ออกจากเซิร์ฟไม่ถูกลบ** — ต่างจากของเดิมที่ลบใน `playerDropped` ซึ่งเป็นสาเหตุที่หลุดเน็ตทีเสียเมล็ดฟรี

**`lp_planting:harvested` เป็น event ให้ resource อื่นเกาะ** — ส่ง `(src, seed, item, count)` ใช้แทน `MJ-Planting:Giveitem` เดิมที่ถูกถอดออกไป
