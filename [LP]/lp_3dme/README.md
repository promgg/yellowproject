# lp_3dme

ข้อความ RP ลอยเหนือหัวผู้เล่น สำหรับ RedM/VORP

## คำสั่ง

| คำสั่ง | สี | ระยะ | ทำอะไร |
|---|---|---|---|
| `/me <ข้อความ>` | เขียว | 20 ม. | บรรยายการกระทำของตัวเอง |
| `/do <ข้อความ>` | น้ำเงิน | 20 ม. | บรรยายสภาพแวดล้อม/สิ่งของ |
| `/say <ข้อความ>` | เขียว | 20 ม. | พูดออกเสียง |
| `/ooc <ข้อความ>` | เทา | 20 ม. | คุยนอกบท |
| `/maskname <ข้อความ>` | เขียว | 20 ม. | แสดงตัวตนที่สวมอยู่ |
| `/mefar <ข้อความ>` | เขียวเข้ม | 50 ม. | เหมือน `/me` แต่คนไกลก็เห็น |
| `/dofar <ข้อความ>` | น้ำเงินเข้ม | 50 ม. | เหมือน `/do` แต่คนไกลก็เห็น |
| `/dice [ลูก] [หน้า]` | น้ำตาล | 20 ม. | ทอยลูกเต๋า ไม่ใส่ = 1 ลูก 6 หน้า |
| `/roll` | เหลือง | 20 ม. | สุ่ม 1-100 |

ข้อความยาวสุด 150 ตัวอักษร ค้าง 10 วินาที (ยกเว้น `/desc` ที่ค้างจนกว่าจะลบ)

## ตั้งค่า

ทุกอย่างอยู่ใน `config.lua` ที่เดียว

เพิ่มคำสั่งใหม่ = เติม entry ใน `Config.Types` — ระบบลงทะเบียนคำสั่งและ chat suggestion
จากตารางนั้นเอง ไม่ต้องแตะโค้ด

## หลักความปลอดภัยที่ยึดไว้

ระบบข้อความลอยแบบนี้พังง่ายกว่าที่คิด — 4 ข้อนี้คือสิ่งที่ต้องรักษาไว้ถ้าจะแก้ต่อ

**1. ไม่มี `innerHTML` ที่ไหนเลย** ข้อความส่งเป็นข้อมูลล้วนแล้วให้ NUI ใส่ด้วย
`textContent` — ถ้าเผลอต่อสตริง HTML ผู้เล่นพิมพ์ `/me <img src=x onerror="...">`
แล้วรันสคริปต์ใน NUI ของทุกคนที่มองเห็นได้ทันที

ทดสอบยิง 4 เพย์โหลดแล้ว แท็กกลายเป็นตัวอักษรธรรมดาทั้งหมด ไม่มี DOM ถูกสร้าง
และอักขระปกติอย่าง `&` `<` `>` ยังแสดงถูกต้อง

**2. server กรองระยะก่อนส่ง** ไม่ใช่ `TriggerClientEvent(..., -1, ...)` แล้วให้ client
กรองเอง — แบบหลังข้อความจะไปถึงทุกคนทั้งแมพ ดักอ่านได้หมด

**3. จำกัดความยาวและ rate limit ฝั่ง server** ตัดที่ 150 ไบต์ (ถอยกลับจนเจอขอบตัวอักษร
UTF-8 ที่ถูกต้อง ไม่ตัดกลางตัวอักษรไทยจนได้ไบต์เสีย) และจำกัด 1.5 วินาที/ข้อความ
15 ข้อความ/นาที

**4. ผลลูกเต๋าสุ่มที่ server** client ส่งมาได้แค่ "ขอทอย" ไม่มีทางกำหนดผลลัพธ์

อีกข้อที่ไม่ใช่ความปลอดภัยแต่สำคัญ: ลูปวาดหลับ 500ms เมื่อไม่มีข้อความ และส่ง NUI
เฉพาะตอนข้อมูลเปลี่ยนจริง — ระบบแบบนี้มักเผลอวิ่งลูปถี่ๆ ตลอดเวลาแม้จอว่าง

## รายละเอียดที่เจอตอนทำ ควรรู้ถ้าจะแก้ต่อ

**กล่องต้องปักที่ `0,0` แล้วเลื่อนด้วย `transform` ล้วน ห้ามใช้ `left`/`top` เป็น %**
ถ้าตั้ง `left: 100%` พื้นที่ที่เหลือทางขวาเป็นศูนย์ เบราว์เซอร์จะบีบกล่องแล้วตัดบรรทัด
ทีละตัวอักษรตั้งแต่ก่อนโค้ดหนีบขอบจอจะทำงาน — วัดจริงแล้วกล่องริมขวาเหลือกว้าง 45px สูง 176px

**`@keyframes` ห้ามแตะ `transform`** เพราะ JS ใช้ transform หนีบกล่องไม่ให้ทะลุขอบจอ
ถ้า keyframes เขียน transform ด้วยจะแย่งกันช่วง 120ms แรกแล้วกล่องกระตุก
animation จึงเหลือแค่ `opacity`

**การหนีบขอบจอทำหลังวางตำแหน่งครบทุกใบ** ไม่ใช่สลับวัด-เขียนทีละใบ เพราะ
`getBoundingClientRect()` บังคับ layout — สลับกันจะเกิด reflow ซ้ำทุกใบ

**native ที่ใช้ ยืนยันว่ามีจริงใน RDR3 ทั้งหมด**
`GetScreenCoordFromWorldCoord` (ใช้จริงใน `lp_airdropteam/core/client.lua:197`) และ
กระดูก `skel_head` (ยืนยันจาก `spooni_spooner/data/rdr3/bones.lua`) — มี fallback ไปใช้
พิกัด entity ถ้าหา bone ไม่เจอ

## ข้อจำกัดที่รู้อยู่

- `/desc` เก็บในหน่วยความจำ server รีสตาร์ทรีซอร์สแล้วหาย
  ถ้าอยากให้ค้างข้ามวัน ตาราง `characters` มีช่อง `character_desc` ว่างอยู่แล้ว
- สถานะ `/desc` ส่งให้ทุกคนแล้วให้ client กรองระยะตอนวาด ต่างจากข้อความอื่นที่กรองฝั่ง server
  เพราะมันเปลี่ยนน้อยมากและเป็นข้อความสั้น การซิงก์ตามระยะแบบ realtime ไม่คุ้มความซับซ้อน
- `/maskname` ตอนนี้เป็นกล่องข้อความธรรมดาสีเดียวกับ `/me` เพราะเซิร์ฟยังไม่มีระบบหน้ากาก
  ถ้าวันหลังมี ให้ผูกกับระบบนั้นแทน
