# lp_leaderboard — เช็คลิสต์เทสในเกม

ก่อนเริ่ม: ใส่ `ensure lp_leaderboard` ใน server.cfg (วางหลัง `vorp_core`, `oxmysql`, `nx_cityselect`, `pNotify`) แล้ว restart เซิร์ฟ
ตรวจ error log ตอน start — ต้องเห็น `[lp_leaderboard][TX] db-loaded kills=.. cities=.. gather=.. (jobs=4)` โดยไม่มี error ก่อน

---

## 0. Boot / เปิด-ปิด UI
- [ ] เซิร์ฟ start ไม่มี error ใน console (โดยเฉพาะ `db-error` — ถ้าเห็นแปลว่าตาราง/DB มีปัญหา)
- [ ] พิมพ์ `/leaderboard` → UI เปิด, เห็น sidebar เหลือ **2 หมวด** (KILL RANK, CITY RANK) — fish/mining/planting/lumber ต้อง **ไม่โชว์** (ปิดไว้ใน config)
- [ ] พิมพ์ `/rank` (alias) → เปิด UI ได้เหมือนกัน
- [ ] กด ✕ หรือ ESC → ปิด UI, คืน control ให้เกม (ไม่ค้าง NuiFocus)
- [ ] เปิด UI ค้างไว้ ~5 วิ โดยไม่มีเหตุการณ์ใดเกิดขึ้น → ไม่มี error, ไม่กระพริบ/รีเฟรชรัว (แปลว่า boardsDirty guard ทำงาน)

## 1. KILL RANK — คำนวณคะแนน
- [ ] ผู้เล่น A ยิงผู้เล่น B ตาย (คนละเมืองกัน, ระยะไม่เกิน 300m) → เปิด `/leaderboard` แล้ว A ควรมี **KILLS +1, SCORE +1**, B ควรมี **DEATHS +1**
- [ ] เปิด UI ค้างไว้ตอนยิง (ไม่ต้องปิด-เปิดใหม่) → ตัวเลขควรอัปเดตเองภายใน ~2.5 วิ (live-push)
- [ ] ตายเอง (ตกน้ำ/ตกที่สูง/ไฟไหม้) → **ไม่มีใครได้ kill**, เหยื่อได้ deaths ตามปกติ
- [ ] ถูก NPC ฆ่าตาย → **ไม่มีใครได้ kill**, เหยื่อได้ deaths ตามปกติ (killerServerId=0)
- [ ] badge เปลี่ยนตาม score: 0=BRONZE, ≥50=SILVER, ≥150=GOLD, ≥400=PLATINUM, ≥1000=DIAMOND (เทสจริงอาจต้องใช้ /addbp-แบบเทียบ หรือยิงสะสมหลายครั้ง)

## 2. KILL RANK — กันโกง (สำคัญสุด)
- [ ] **same-city guard**: ผู้เล่น 2 คนเลือกเมืองเดียวกัน (เช่น Valentine ทั้งคู่ผ่าน nx_cityselect) ยิงกันตาย → **ไม่มีใครได้ kill, เหยื่อก็ไม่ได้ deaths** (ตั้งใจแบบนี้ตามที่ตกลง — ยืนยันว่ายังต้องการ behavior นี้)
- [ ] **farm-cooldown 90 วิ**: A ฆ่า B แล้วรีบ respawn มาให้ A ฆ่าซ้ำภายใน 90 วิ → ครั้งที่ 2 **ไม่ได้ kill เพิ่ม** (เช็ค log `farm-guard` ถ้าเปิด `Config.Debug=true`)
- [ ] รอเกิน 90 วิแล้วให้ A ฆ่า B คู่เดิมอีกครั้ง → **ได้ kill ตามปกติ**
- [ ] **maxKillDistance 300m**: ถ้าทำได้ (เช่นใช้ noclip/ลากตัวละครออกไปไกลๆ ทดสอบ) — ยิงคนที่อยู่ไกลเกิน 300m แล้ว "ตาย" (ผ่านทาง client แปลกๆ) ไม่ควรนับ kill — ปกติทดสอบยาก ข้ามได้ถ้าไม่มีเครื่องมือ
- [ ] ตายรัว ๆ ภายใน 1 วิ (เช่นโดนยิงซ้ำ) → เหยื่อได้ deaths แค่ครั้งเดียว (spam-guard 1 วิ/เหยื่อ)

## 3. CITY RANK — ผ่าน lp_airdropteam
- [ ] เริ่มรอบ airdrop, ผู้เล่นจากเมือง A, B, C เข้าร่วม (คนละ NPC สถานีก็ได้ ขอแค่ battle team ผูกเมืองต่างกัน) → ให้คนเมือง A ลูทกล่องสำเร็จก่อน
- [ ] จบรอบ (AirdropCount=0 หรือหมดเวลา) → เปิด `/leaderboard` แท็บ CITY RANK: เมือง A,B,C ควรมี **เข้า +1** ทั้งหมด, เมือง A มี **ชนะ +1**, เมือง B/C มี **แพ้ +1**
- [ ] ชนะ% คำนวณถูก (wins/(wins+losses)*100 ปัดเศษ)
- [ ] รอบที่ไม่มีใครลูทสำเร็จเลย (หมดเวลา ไม่มีผู้ชนะ) → ทุกเมืองที่เข้าร่วมได้แค่ **เข้า +1** ไม่มีใครได้ชนะ/แพ้ (winner=nil)
- [ ] รอบที่มีแค่เมืองเดียวเข้าร่วม แล้วเมืองนั้นชนะ → เมืองนั้น เข้า+1, ชนะ+1, ไม่มีเมืองแพ้เลย

## 4. YOUR RANKING banner
- [ ] เปิด UI แล้วยังไม่เคย kill/death เลย → เห็นแถบ "YOUR RANKING #-" (rank เป็น "-" ไม่ crash)
- [ ] มีสถิติแล้ว → เห็นอันดับตัวเองถูกต้อง, ชื่อ, badge, SCORE/KILLS/DEATHS ตรงกับข้อมูลจริง
- [ ] แถวตัวเองในตารางหลัก ต้องถูกไฮไลต์ (กรอบทอง/สีต่าง) ตรงกับ rank เดียวกับ banner
- [ ] แท็บ CITY RANK: ถ้าตัวเองยังไม่เคยเลือกเมือง (nx_cityselect ยังไม่ตั้งค่า) → banner ควรไม่ crash (คน viewer.cityId เป็น nil ได้)

## 5. Reset (admin only)
- [ ] ผู้เล่น **ไม่ใช่ admin** เปิด UI → ปุ่ม "RESET SCORE" ต้อง **ไม่แสดง** เลย
- [ ] ผู้เล่นกลุ่ม `admin`/`superadmin` เปิด UI → เห็นปุ่ม RESET SCORE
- [ ] กดปุ่มครั้งแรก → ข้อความเปลี่ยนเป็น "กดอีกครั้งเพื่อยืนยัน" (ยังไม่รีเซ็ต)
- [ ] ไม่กดซ้ำภายใน 3 วิ → ปุ่มคืนข้อความเดิมอัตโนมัติ (ไม่รีเซ็ต)
- [ ] กดซ้ำภายใน 3 วิ (ยืนยัน) → เฉพาะ**แท็บที่เปิดอยู่ตอนนั้น**ถูกรีเซ็ต (เช่นอยู่แท็บ KILL ก็รีเซ็ตแค่ kill ไม่โดน city) + มี toast "รีเซ็ตกระดานอันดับเรียบร้อย"
- [ ] รีเซ็ตแล้ว เปิด UI ใหม่/รอ live-push → ตัวเลขในแท็บนั้นกลับเป็น 0/ว่างจริง
- [ ] ลองยิง `reset` event ถี่ ๆ ติดกัน (เช่นกดปุ่มซ้ำเร็วมาก ผ่าน dev console) → ต้องเจอ cooldown 2 วิ ไม่ทำงานซ้อน
- [ ] เช็ค log: มีบรรทัด `admin-reset by=<ชื่อ>(<charid>) category=<...>` ทุกครั้งที่รีเซ็ตสำเร็จ

## 6. Rate limit (server)
- [ ] สแปม `/leaderboard` รัว ๆ (กดหลายครั้งเร็ว ๆ ภายใน 1 วิ) → ไม่มี error, ไม่เห็นอาการหน่วง/ค้าง (cooldown 1000ms กันไว้)
- [ ] ปิด-เปิด UI สลับไปมาเร็ว ๆ หลายรอบ → resource ไม่ crash, NuiFocus ไม่ค้าง

## 7. Live-push ระหว่างเปิดค้าง
- [ ] เปิด UI ค้างไว้ที่แท็บ KILL RANK แล้วให้ผู้เล่นอื่นไปฆ่าคนตาย (นับได้) → ตัวเลข/อันดับใน UI ที่เปิดค้างอยู่ต้อง**อัปเดตเอง**ภายใน ~2.5 วิ โดยไม่ต้องปิดเปิดใหม่
- [ ] ผู้เล่นที่ **ไม่ได้เปิด UI** ตอนนั้น ไม่ควรได้รับ SendNUIMessage ใด ๆ (เช็คผ่าน log/perf — ไม่มีทางเทสตรง ๆ ในเกม แต่ถ้า client ไม่มีอาการกระตุกก็ถือว่าโอเค)

## 8. Reconnect / cleanup
- [ ] ผู้เล่นเปิด UI ค้างไว้แล้ว disconnect (ปิดเกมตรง ๆ ไม่กด ✕) → เซิร์ฟไม่ error, ผู้เล่นคนอื่นที่เปิด UI อยู่ยังใช้งานได้ปกติ
- [ ] ผู้เล่นคนเดิม connect กลับมาใหม่ → เปิด `/leaderboard` ได้ปกติ ไม่มี state ค้างจากรอบก่อน
- [ ] restart resource (`restart lp_leaderboard`) ระหว่างมีคนเปิด UI ค้าง → ไม่ error รุนแรง, เปิดใหม่ได้หลัง resource กลับมา, สถิติ (kills/deaths/city) ยังอยู่ครบ (โหลดจาก DB ถูก)

## 9. Performance / สังเกตทั่วไป
- [ ] เปิด `Config.Debug = true` ชั่วคราวระหว่างเทส เพื่อดู log `kill:`, `farm-guard:`, `same-city guard:`, `distance-guard:` ตอนเกิดเหตุการณ์จริง แล้วปิดกลับหลังเทสเสร็จ
- [ ] `resmon`/txAdmin profiler เช็คว่า resource ไม่กิน CPU ผิดปกติตอนไม่มีคนเปิด UI (ควรใกล้ 0 — ไม่มี per-player thread)
- [ ] เล่นต่อเนื่อง 10+ นาทีมีคนตาย-เกิดหลายรอบ → เช็คว่า memory ไม่พุ่งผิดปกติ (เช็คคร่าว ๆ ผ่าน txAdmin หรือ resmon)

## เตรียมก่อนเปิดหมวด Gather (fish/mining/planting/lumber) ในอนาคต
เมื่อพร้อมเทสหมวดเก็บของ ให้เปลี่ยน `enabled = false` → `true` ใน `Config.Categories` (config.lua บรรทัด 109-112) ทีละหมวด แล้วเช็คเพิ่ม:
- [ ] ตกปลา/ขุด/ปลูก/ตัดไม้สำเร็จ 1 ครั้ง → SCORE ขึ้นตามจำนวนไอเทมที่ได้จริง (ไม่ใช่แค่ +1), count (CATCHES/DIGS/HARVESTS/CHOPS) ขึ้น +1
- [ ] ของเต็มกระเป๋าจนได้ไอเทม 0 ชิ้น (เช่น fishing inventoryFull) → **ไม่ควร**นับเข้าบอร์ด (ต้นทางเช็ค `addItem` สำเร็จก่อนยิง event แล้ว)
- [ ] badge ของแต่ละหมวดขึ้นตาม threshold เฉพาะหมวดนั้น (ตัวเลขต่างจาก KILL RANK)
